1. What are some differences between interfaces and types in TypeScript?

উত্তরঃ interfaces এবং types এর ভেতর কিছু পার্থক্য আছে , নিচে এগুলো ব্যাখ্যা করা হলো : 

interfaces মুলত ব্যাবহার করা হয় object এর জন্য এবং এটা এক্সটেনডবল, এটা interface কিওয়ারড ব্যাবহার করে। একই নাম এর দুটি interface মারজ হয়ে যায়। interface ইউনিয়ন টাইপ  ইমপ্লিমেন্ট এবং এক্সটেন্দিং সাপোর্ট করে। interface ইন্টারসেকশন টাইপ তৈরি করতে পারে না।

type ব্যবহার করা হয় কমপ্লেক্স টাইপ ডেফিনেশনের জন্য, type বেশি ভারসেটাইল। এটা type কিওয়ারড ব্যাবহার করে। এটা object সাপোর্ট করে না। একই নাম এর দুটি type এক্সেপ্সন দেয়। type ইউনিয়ন টাইপ  ইমপ্লিমেন্ট এবং এক্সটেন্দিং সাপোর্ট করে না। type ইন্টারসেকশন টাইপ তৈরি করতে পারে।


2. Explain the difference between any, unknown, and never types in TypeScript.

উত্তরঃ any, unknown এবং never এর ভেতর কিছু পার্থক্য আছে , নিচে এগুলো ব্যাখ্যা করা হলো : 

any এর মাধ্যমে টাইপস্ক্রিপ্টে টাইপ ফাংশন বন্ধ করা যায় তাই কোন ভ্যারিয়েবল টাইপ থাকে না।
any সাধারণত জাভাস্ক্রিপ্ট থেকে টাইপিস্ক্রিপ্টে মাইগ্রেট করার জন্য ব্যবহৃত হয়। 

unknown , any এর মত কাজ করে কিন্তু unknown এর ভিতরে যে কোন টাইপ সাইন করা যায়, আমরা unknown তাই চেক না করে ব্যবহার করতে পারি না। আমরা unknown ইউজ করি যখন আমরা জানি না যে কোন টাইপ ইউজ করতে হবে এবং আমরা যখন এপিআই অথবা ইউজার ইনপুট অথবা জেসন থেকে ডাটা নেই তখন আমরা unknown ইউস করি এছাড়াও আমরা যখন সেফটি এবং ফ্লেক্সিবিলিটি চাই তখন আমরা unknown ইউজ করি।

never ইউজ করা হয় সে সকল ভ্যালুর জন্য যেখানে  টাইপ এক্সিসট করে না এবং সে সকল ফাংশানের ক্ষেত্রে যেগুলো কোন ভ্যালু রিটার্ন করে না। সুইচ কেস এর ক্ষেত্রে এটি নিশ্চিত করে যে সকল পসিবল কেস হ্যান্ডেল করা হয়েছে।
